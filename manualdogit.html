    O que é Git

    Git é um sistema de controle de versão de código utilizado por muitas empresas, existem outros mais ultrapassados, mas os sistemas mais atuais são baseados em git. Algumas empresas são especializadas em fornecer os serviços para o controle de histórico de alterações do código, e diversas outras empresas que utilizam dessas ferramentas para manter a consistência na equipe de desenvolvimento.

    configuração


    Para configurar o nome e o email associados à sua conta do GitHub e vinculá-los localmente, use os comandos corretos:

    • git config --global user.name "[seu nome
    • git config --global user.email "[seu endereço de email]"

    Git init e Git add .

    • O comando git init é usado para criar um repositório local.
    • O comando git add é utilizado para adicionar arquivos à área de preparação. git add . adicionará todos os arquivos modificados ou novos ao próximo commit

    Git commit

    • O comando git commit -m "mensagem" é usado para confirmar suas alterações na área de preparação com uma mensagem de commit. Não é necessário mencionar que ele adiciona arquivos, pois isso é feito anteriormente com o comando git add.
    • A explicação sobre git reset HEAD~1 está correta. Ele é usado para voltar um commit, desfazendo o commit mais recente.

    Git log e Git status

    git log é usado para verificar o histórico de commits, não o histórico de ramificações. git status é utilizado para verificar o status dos arquivos em relação à área de preparação e ao diretório de trabalho. Git branch: O comando para listar todas as ramificações é git branch, mas para excluir uma ramificação, use git branch -d [nome da ramificação]. git checkout é usado para alternar entre ramificações.

    Git remote

    • O comando para adicionar um vínculo a um repositório remoto é git remote add origin [endereço remoto].
    • remote -v é usado para visualizar as informações sobre os repositórios remotos conectados ao repositório local.
    • Para remover o vínculo a um repositório remoto, use git remote rm origin. Para renomear um repositório remoto, use git remote rename "nome atual" "nome novo".

      • Git push e Git pull

      • O comando git push é usado para enviar as alterações locais para o repositório remoto, não para puxar da máquina local para a remota.
      • O comando git pull é usado para puxar as alterações do repositório remoto para a máquina local.

      Git ignore

      • Especifica arquivos e diretórios que devem ser ignorados pelo Git. Crie um arquivo .gitignore e liste os padrões de arquivos/diretórios a serem ignorados.
